# ThreatLens benchmark

Synthetic JSON corpus generated by `scripts/generate_sample_data.py`, scanned
with built-in detectors only (no Sigma layer). Numbers from a Windows 11 host,
Python 3.11.9, on a single core - the scanner is not yet parallelised.

| Events | File size | Parse | Detect | Total | Throughput | Alerts |
|--------|-----------|-------|--------|-------|------------|--------|
|   9,009 |   2.3 MB |  0.09s |  0.04s |  0.13s |  69.3 k/s |  1,009 |
|  90,145 |  22.6 MB |  0.89s |  0.38s |  1.27s |  71.0 k/s | 10,534 |
| 901,341 | 226.2 MB |  9.90s |  4.32s | 14.24s |  63.3 k/s | 96,424 |

Reproduce locally:

```bash
python scripts/benchmark.py --sizes 10000 100000 1000000 --out docs/benchmark_results.md
```

## Notes

- Alert counts are inflated relative to a real-world workload because the
  synthetic generator embeds an attack chain every five thousand events.
- The detection-phase cost is dominated by the attack-chain correlator, which
  walks events twice. A future refactor that streams correlations should pull
  this down by roughly half.
- Sigma rule evaluation is excluded from these numbers. Add `--sigma-rules
  rules/sigma_starter` to the scan command for a comparable measurement that
  layers the bundled rule pack on top.
